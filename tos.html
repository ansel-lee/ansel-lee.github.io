i. Google login

I use google authentication in my web app so that I can identify who submitted my form, as the web is a big space and there are many people, some with malicious intents, who can view my site. The only information that I request at this step is your email, name and profile picture.

The token from the sign in is then sent to my backend, which is a google apps script project which will generate a long key string that I will use for your identification later on. This key will tell me which column inside my google sheet will be used to record your answer. I use a key instead of a column name to prevent users with malicious intent impersonating someone, entering information that will shame that person, and then there will be a big problem. 

ii. On question submit

Every time you submit a question, I will record your browser version and device, and cross-check it with the information already inside the sheet. my app will alert you should someone on another device try to send a request to the server with your key. Unless you have some superpower ability to be able to transfer the key (there is no text field to input the key inside the form), you should pick the option of creating a new key.

d. Google forms

I think almost nothing except the form data is collected. Just a reminder to not post anything that is private or inappropriate.
